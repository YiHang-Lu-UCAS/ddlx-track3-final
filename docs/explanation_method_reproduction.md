# Method-Level Reproduction of Text Explanations

The purpose of this section is to show that `Visible forgery traces` was generated by a model, not manually written.

## Explanation model

The bundle includes the full explanation model stack:

```text
models/explanation/qwen2_5_vl_3b_instruct/
models/explanation/qwen2_5_vl_3b_lora_checkpoint1500/
```

The model call used by the reproduction scripts is:

```bash
swift infer \
  --model models/explanation/qwen2_5_vl_3b_instruct \
  --adapters models/explanation/qwen2_5_vl_3b_lora_checkpoint1500 \
  --template qwen2_5_vl \
  --infer_backend pt \
  --torch_dtype float16 \
  --max_length 4096 \
  --max_pixels 602112 \
  --max_new_tokens "$QWEN_MAX_NEW_TOKENS" \
  --temperature 0 \
  --num_beams 1
```

## Reproduce explanations for a submission JSON directory

Given JSON files whose `Classification result` and `Bounding boxes` have already been produced by the classifier/localizer, run:

```bash
bash scripts/run_explanation_inference.sh \
  /path/to/submission_json_or_root \
  /path/to/test_images.csv \
  /path/to/output_qwen_explanations \
  6 \
  0,1,2,3,4,5
```

This script performs the full explanation method:

1. Builds Qwen/VL JSONL input records from images, labels, and boxes.
2. Calls Qwen2.5-VL-3B-Instruct + checkpoint-1500 LoRA with `swift infer` using `QWEN_MAX_NEW_TOKENS=2048` by default.
3. Replaces only the `Visible forgery traces` field with model responses.
4. Writes regenerated JSON files and a zip.
5. Emits a SHA256 for the generated zip.

## Reproduce fake-nobox fallback text rerun

The selected final WBF submission uses the `fake_nobox_nose_eyes_mouth` variant. To reproduce the method used for fake images where the normal variant had no retained detector box:

```bash
bash scripts/run_fake_nobox_explanation_rerun.sh \
  /path/to/source_submission_root_with_variants \
  /path/to/test_images.csv \
  /path/to/output_fake_nobox_qwen_rerun \
  6 \
  0,1,2,3,4,5
```

## Role of `text_sources/`

`text_sources/` is included only as an exact cache/provenance artifact for rebuilding the submitted leaderboard zip. It is not the method. The method is the Qwen base model, LoRA checkpoint, prompt construction scripts, and `swift infer` commands above. Detailed generation parameters are in `docs/explanation_generation_details.md`.
